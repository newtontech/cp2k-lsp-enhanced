# CP2K LSP Enhanced - New Features

## Overview

This enhanced version of the CP2K Language Server Protocol implementation includes advanced validation and completion features based on CP2K's official schema.

## New Features

### 1. CP2K CLI Deep Validation

Integrates CP2K's native syntax checking (`cp2k -c`) with LSP diagnostics.

**Features:**
- Real-time syntax validation using CP2K's own parser
- Captures errors, warnings, and information from CP2K output
- Parses CP2K error messages into LSP diagnostics with proper locations
- Debounced validation to avoid excessive CP2K invocations
- Configurable timeout and executable path

**Configuration (VSCode settings.json):**
```json
{
  "cp2k.enableDeepValidation": true,
  "cp2k.cp2kPath": "/path/to/cp2k.psmp",
  "cp2k.validationDelay": 1000
}
```

**Enabling Deep Validation:**
1. Ensure CP2K is installed and in your PATH
2. Set the correct path to CP2K executable
3. Enable `enableDeepValidation` in your configuration
4. Save your CP2K input file - diagnostics will appear automatically

### 2. XML Schema Parser

Parses CP2K's official XML schema exported by `cp2k --xml` command.

**Features:**
- Automatic schema generation from CP2K
- Schema caching for fast startup
- Comprehensive extraction of:
  - Sections and subsections
  - Keywords with all metadata
  - Data types (STRING, INTEGER, REAL, LOGICAL, ENUM, etc.)
  - Allowed values for enums
  - Default values
  - Units
  - Repetition flags
  - Deprecated status
  - Required flags
  - Mutually exclusive constraints

**Schema Generation:**
```bash
# Manually generate schema (automatic on first use)
cp2k --xml > cp2k-schema.xml

# The LSP will automatically generate and cache the schema
```

**Schema Cache:**
- Located at: `data/cp2k-schema-cache.json`
- Automatically regenerated if cache is outdated or missing
- Can be manually regenerated by deleting the cache file

### 3. Enhanced Diagnostics

Comprehensive validation beyond basic syntax checking.

**Validation Types:**

1. **Required Sections:**
   - Detects missing `GLOBAL` section
   - Detects missing `FORCE_EVAL` section
   - Detects required subsections

2. **Required Keywords:**
   - Checks for required keywords in sections
   - Identifies missing mandatory parameters

3. **Type Checking:**
   - Integer validation
   - Real number validation
   - Logical value validation (TRUE/FALSE/YES/NO)
   - List type validation

4. **Constraint Validation:**
   - Mutually exclusive keyword detection
   - Required keyword/section combinations
   - Repetition constraints

5. **Semantic Validation:**
   - Variable expansion syntax (`${VAR}`)
   - Balanced parentheses and brackets
   - Valid section and keyword names

6. **Deprecated Features:**
   - Warns about deprecated sections
   - Warns about deprecated keywords

**Diagnostic Codes:**
- `missing-section`: Required section missing
- `missing-keyword`: Required keyword missing
- `missing-subsection`: Required subsection missing
- `unknown-keyword`: Keyword not found in schema
- `invalid-value`: Value doesn't match allowed values
- `type-mismatch`: Value doesn't match expected type
- `mutual-exclusion`: Mutually exclusive keywords present
- `deprecated-section`: Section is deprecated
- `deprecated-keyword`: Keyword is deprecated

### 4. Enhanced Completion

Intelligent code completion based on official CP2K schema.

**Completion Features:**

1. **Section Completion:**
   - Trigger: Type `&` or section name prefix
   - Snippets with `&END` tags
   - Context-aware section suggestions

2. **Keyword Completion:**
   - Trigger: At line start or after section header
   - Section-specific keywords
   - Default value snippets
   - Type information

3. **Value Completion:**
   - Enum values (e.g., RUN_TYPE → ENERGY, GEOMETRY_OPTIMIZATION, etc.)
   - Boolean values (TRUE/FALSE/.TRUE./.FALSE./YES/NO)
   - Context-aware suggestions

4. **Unit Completion:**
   - Trigger: After numbers (space + prefix)
   - Categories:
     - Length: angstrom, bohr, nm, pm, m
     - Energy: hartree, eV, kcalmol, kJmol, Ry, J
     - Time: fs, ps, s
     - Temperature: K
     - Pressure: bar, atm, Pa, GPa
     - Mass: amu
     - Angle: deg, rad
     - Force: hartree/bohr

5. **Context Awareness:**
   - Knows current section
   - Provides relevant suggestions only
   - Handles nested sections correctly

**Completion Examples:**

```cp2k
# Section completion
&GLO  →  GLOBAL

# Keyword completion
&GLOBAL
  PRO  →  PROJECT_NAME TEST

# Enum value completion
  RUN_TYPE ENER  →  ENERGY

# Unit completion
  CUTOFF 400 an  →  angstrom

# Boolean completion
  MAP_CONSISTENT  →  TRUE, FALSE, .TRUE., .FALSE.
```

## Usage Guide

### Basic Setup

1. **Install the language server:**
   ```bash
   npm install
   npm run build
   ```

2. **Configure VSCode:**
   ```json
   {
     "cp2k.enableSchemaValidation": true,
     "cp2k.enableDeepValidation": false,
     "cp2k.cp2kPath": "",
     "cp2k.maxNumberOfProblems": 100
   }
   ```

### Advanced Configuration

**Enable Schema Validation:**
```json
{
  "cp2k.enableSchemaValidation": true
}
```
- Fast, doesn't require CP2K executable
- Uses cached schema
- Provides type checking and constraints

**Enable Deep Validation:**
```json
{
  "cp2k.enableDeepValidation": true,
  "cp2k.cp2kPath": "/usr/local/bin/cp2k.psmp",
  "cp2k.validationDelay": 1000
}
```
- Uses CP2K's native parser
- Catches all CP2K errors
- Slower (requires external process)
- Requires CP2K installation

**Both Enabled:**
```json
{
  "cp2k.enableSchemaValidation": true,
  "cp2k.enableDeepValidation": true
}
```
- Best of both worlds
- Fast schema validation (immediate)
- Deep validation (debounced, after typing stops)
- Comprehensive error coverage

## Performance

- **Schema validation:** < 10ms (in-memory)
- **Deep validation:** 100-500ms (depends on CP2K, debounced)
- **Completion:** < 5ms (cached lookup)

## Troubleshooting

**Schema not loading:**
- Check CP2K path in settings
- Verify CP2K supports `--xml` flag
- Delete cache: `rm data/cp2k-schema-cache.json`

**Deep validation not working:**
- Verify `cp2kPath` is correct
- Check CP2K has `-c` flag support
- Look at LSP server logs

**Completions not appearing:**
- Check schema is loaded
- Verify trigger characters: `&`, ` `, `\n`, `_`, `/`
- Check VSCode completion settings
